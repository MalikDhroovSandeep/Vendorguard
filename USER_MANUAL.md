5.3 User Manual with Screenshots


Part A: Authentication

1] Landing Page, Sign Up, and Login – Open a browser and navigate to the application URL. The Landing Page displays the VendorGuard branding, hero section, feature cards, and role descriptions. Click "Sign Up" to create an account by entering Full Name, Email, Password, and selecting a role (Admin / Internal User / Vendor). Social sign-up via Google, Facebook, or Apple is also available. To login, enter Email and Password on the Login page and click "Sign in". Use "Forgot password?" to reset via email if needed.

Part B: Internal User and Vendor Workflow

2] Internal User Dashboard and Vendor Onboarding – After logging in as Internal User, the dashboard shows KPI cards (Vendor Count, Active Orders, Pending Invoices, Open Disputes), alert badges, and a Recent Activity table. The Internal User can add a new vendor by navigating to Vendors and clicking "Add New Vendor" with business details. The vendor then signs up, and on the Vendor Dashboard sees a "Complete KYC" banner. Vendor navigates to KYC, fills in GST number, PAN number, business document details, uploads supporting files, and clicks "Submit".

[Screenshot: Internal User Dashboard]
[Screenshot: Vendor KYC Submission Form]

3] Purchase Orders and Delivery – Internal User navigates to Orders and clicks "Create Order". Selects a vendor (risk level displayed beside each name), enters order title, items, quantities, and amounts. After submission, the order appears in the table with status tracking. The Vendor views the assigned order on the Vendor Orders page and updates delivery status (Partial / Completed / Delayed).

[Screenshot: Create Order Modal]
[Screenshot: Vendor Orders Page]

4] Invoices – Vendor navigates to Invoices, clicks "Create Invoice", selects the linked order, enters amount, tax, and due date. Internal User reviews submitted invoices and can Approve, Reject, or Mark as Paid.

[Screenshot: Invoices Page]

5] Disputes and Returns – Internal User raises a dispute by linking it to an order, selecting category and priority, and entering a description. Vendor receives notification and submits a one-time response. For returns, Internal User clicks "Create Return", selects the order, adds return items with quantity and reason.

[Screenshot: Disputes Page]
[Screenshot: Returns Page]


Part C: Admin Oversight and AI Analytics

6] Admin Dashboard and KYC Verification – After logging in as Admin, the dashboard shows KPI cards (Total Vendors, Pending KYC, High Risk Vendors, Open Disputes), Risk Distribution Chart, Vendor Status Breakdown, and Recent Vendors table. Admin navigates to KYC to review vendor-uploaded documents and clicks "Approve" or "Reject" with a reason.

[Screenshot: Admin Dashboard]
[Screenshot: Admin KYC Page]

7] Dispute Resolution and Return Approval – Admin opens a dispute, reviews the vendor response, updates status (Under Review / Resolved / Closed), and adds resolution notes. For returns, Admin approves with a claim type (Refund / Replacement / Credit Note) and a credit note is auto-generated.

[Screenshot: Admin Dispute Resolution]

8] AI Risk Analytics and Vendor Performance – Admin navigates to Risk Analytics to view risk score distribution, anomaly alerts with severity levels, and AI performance predictions. Vendors can view their own Performance page showing a risk score gauge, sub-scores (Delivery, Dispute, Payment, Fulfilment), trend chart, and improvement recommendations.

[Screenshot: Admin AI Risk Analytics]
[Screenshot: Vendor Performance Page]

9] Reports, Notifications, and Settings – Admin navigates to Reports to export data in JSON, PDF, or Excel. Audit Logs show a chronological trail of all system actions. The Notification Centre (bell icon) delivers alerts for all key events. Settings page allows profile updates, password changes, and notification preferences for all roles.
